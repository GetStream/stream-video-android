[Control Actions](../04-ui-components/04-call/04-call-controls.mdx) represents the set of controls the user can use to change their audio and video device state, or browse other types of settings, leave the call, or implement something custom.
Stream UI components, such as [CallContainer](../04-ui-components/04-call/01-call-container.mdx), [CallContent](../04-ui-components/04-call/03-call-content.mdx), and [CallLobby](06-lobby-preview.mdx) provide default control actions like the below:

![Call Controls](../assets/cookbook/replacing-call-controls.png)

### Customization

You can simply custom the control actions by building your own list of actions like the below:

```kotlin
val actions = listOf(
    {
        ToggleCameraAction(
            modifier = Modifier.size(52.dp),
            isCameraEnabled = isCameraEnabled,
            onCallAction = { .. }
        )
    },
    {
        ToggleMicrophoneAction(
            modifier = Modifier.size(52.dp),
            isMicrophoneEnabled = isMicrophoneEnabled,
            onCallAction = { .. }
        )
    },
    {
        FlipCameraAction(
            modifier = Modifier.size(52.dp),
            onCallAction = { .. }
        )
    },
    {
        LeaveCallAction(
            modifier = Modifier.size(52.dp),
            onCallAction = { .. }
        )
    },
)
```

Now, you can replace the default control actions with your own actions buttons like the sample below:

```kotlin
CallContainer(
    modifier = Modifier.background(color = VideoTheme.colors.appBackground),
    callControlsContent = {
        ControlActions(
            callViewModel = vm,
            actions = actions
        )
    },
    ..
)
```

With the same approach, you can also change the control actions for `CallContent` and `CallLobby`:

```kotlin
// replace default call actions for CallContent
CallContent(
    callControlsContent = {
        ControlActions(
            callViewModel = vm, // or callDeviceState
            actions = actions
        )
    },
    ..
)

// replace default call actions for CallLobby
CallLobby(
    lobbyControlsContent = {
        ControlActions(
            callViewModel = vm, // or callDeviceState
            actions = actions
        )
    },
    ..
)
```

If you build with your project with the custom control actions above, you will see the result below:

![Compose CallContainer](../../assets/compose_call_controls_custom.png)

### Replacing the control actions

This example shows how to build your own call controls by implementing your own composable:

```kotlin
@Composable
fun MyControlActions() {
    Row(
        modifier = Modifier
            .fillMaxWidth()
            .height(60.dp)
            .padding(top = 10.dp),
        verticalAlignment = Alignment.CenterVertically,
        horizontalArrangement = Arrangement.Center
    ) {
        Icon(
            modifier = Modifier
                .height(40.dp)
                .weight(1f)
                .background(VideoTheme.colors.primaryAccent, CircleShape)
                .padding(10.dp)
                .clickable { vm.onCallAction(ToggleCamera(false)) },
            painter = painterResource(id = R.drawable.ic_video),
            tint = Color.White,
            contentDescription = null
        )

        Icon(
            modifier = Modifier
                .height(40.dp)
                .weight(1f)
                .background(VideoTheme.colors.errorAccent, CircleShape)
                .padding(10.dp)
                .clickable { vm.onCallAction(LeaveCall) },
            painter = painterResource(id = R.drawable.ic_call_end),
            tint = Color.White,
            contentDescription = null
        )
    }
}
```

Next, you can replace the default control actions with your own composable function.

```kotlin
CallContainer(
    modifier = Modifier.background(color = VideoTheme.colors.appBackground),
    callControlsContent = { MyControlActions() },
    ..
)
```

Now, you will see the result below:

![Replaced Control Actions](../../assets/control-actions-replaced.png)


### Removing the control actions

If you want to remove the control actions and render video within full-size, you can just pass an empty lambda to the `callControlsContent` parameter like the sample below:

```kotlin
CallContainer(
    modifier = Modifier.background(color = VideoTheme.colors.appBackground),
    callControlsContent = { },
    ..
)
```

After building the sample code, you'll see the result below:

![Control Actions Empty](../../assets/control-actions-empty.png)
